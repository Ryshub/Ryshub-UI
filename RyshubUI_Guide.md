# RyshubUI Guide

Compact reference for `RyshubUi.luau`.

## Loader

```lua
local Ryshub = loadstring(game:HttpGet("https://cdn.jsdelivr.net/gh/Ryshub/Ryshub-UI@main/RyshubUi.luau"))()
```

## Panda Example

```lua
local Ryshub = loadstring(game:HttpGet("https://cdn.jsdelivr.net/gh/Ryshub/Ryshub-UI@main/RyshubUi.luau"))()

local function runMainScript()
    print("main script executed")
end

Ryshub.Appearance.Title = "RysHub"
Ryshub.Appearance.Subtitle = "Enter your key to continue"
Ryshub.Links.Discord = "https://discord.gg/tuinvite"
Ryshub.Options.ShowGetKeyButton = false
Ryshub.Options.ShowUserPanel = true
Ryshub.Options.StreamerMode = true

Ryshub.Callbacks.OnSuccess = function()
    runMainScript()
end

Ryshub.Callbacks.OnFail = function(msg)
    warn("Validation failed:", msg)
end

Ryshub:LaunchPanda({
    ServiceID = "ryshub"
})
```

Panda keyless example:

```lua
Ryshub.Options.Keyless = true
Ryshub.Options.KeylessUI = false

Ryshub:LaunchPanda({
    ServiceID = "ryshub"
})
```

Security note:

- do not expose the direct URL of your main script in a public `OnSuccess` example
- if users can see that URL, they can copy it and run the main payload directly, bypassing the key UI entirely
- if you need to protect the distributed payload, use an obfuscation/protection layer such as `Luraph` to raise the bar against simple URL stealing and reuse
- obfuscation helps with protection, but it is not a perfect security boundary by itself

## Storage

```lua
Ryshub.Storage = {
    FileName = "RysHubPremiumKey",
    Remember = true,
    AutoLoad = true
}
```

- `LaunchPanda()` does not require `FileName`
- if you do not define a custom one, it uses `ServiceID` as the base for the saved file
- use the base name without `.txt`
- valid example: `"RysHubPremiumKey"`
- actual saved key path: `Ryshub/keys/<FileName>.txt`

## Links

```lua
Ryshub.Links = {
    GetKey = "",
    Discord = "https://discord.gg/tuinvite"
}
```

Notes:

- in Panda, `GetKey` is autogenerated if you do not define it
- if `ShowGetKeyButton = false`, the button is hidden
- for remote loading, `jsDelivr` is the recommended URL in this repo

## Options

```lua
Ryshub.Options = {
    Keyless = nil,
    KeylessUI = false,
    Blur = true,
    Draggable = true,
    ShowUserPanel = true,
    ShowGetKeyButton = true,
    StreamerMode = false
}
```

- `ShowUserPanel = false`: hides the side user panel
- `ShowGetKeyButton = false`: hides the `Get Key` button
- `StreamerMode = true`: masks the player's visible name

## Callbacks

```lua
Ryshub.Callbacks = {
    OnVerify = nil,
    OnSuccess = nil,
    OnFail = nil,
    OnClose = nil
}
```

- `OnSuccess`: not required, but it usually loads your main script
- `OnFail`: optional
- `OnClose`: optional
- `OnVerify`: used by `Launch()`, not by `LaunchPanda()`

## Methods

```lua
Ryshub:Launch()
Ryshub:LaunchJunkie({
    Service = "your-service",
    Identifier = "your-identifier",
    Provider = "your-provider"
})
Ryshub:LaunchPanda({
    ServiceID = "ryshub"
})
```

- `Launch()`: custom validation using `Callbacks.OnVerify`
- `LaunchJunkie()`: validation through the Junkie SDK
- `LaunchPanda()`: validation through PandaDevelopment
- `LaunchPanda()` also accepts `FileName` and `StorageFileName` as optional overrides
- `LaunchPanda()` supports `Options.Keyless` and `Options.KeylessUI`

## Helpers

```lua
local savedKey = Ryshub:GetSavedKey()
Ryshub:ClearSavedKey()
```

## Notes

- this repo is focused on the current Panda flow
- `LaunchJunkie()` is available for compatibility with the original Arqel flow
- `example_panda.lua` is the recommended Panda starting point
- `example_junkie.lua` is the ready-to-use Junkie example
