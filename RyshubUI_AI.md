# RyshubUI AI Reference

Compact document for AI/agents. Functional source of truth: `RyshubUi.luau`.

## Purpose

- Luau keysystem UI
- main repo flow: PandaDevelopment
- relevant public methods:
  - `Ryshub:Launch()`
  - `Ryshub:LaunchJunkie(config)`
  - `Ryshub:LaunchPanda(config)`
  - `Ryshub:Notify(title, message, duration, iconType)`
  - `Ryshub:GetSavedKey()`
  - `Ryshub:ClearSavedKey()`

## Loader

```lua
local Ryshub = loadstring(game:HttpGet("https://cdn.jsdelivr.net/gh/Ryshub/Ryshub-UI@main/RyshubUi.luau"))()
```

## Public Config

### `Ryshub.Appearance`

```lua
Ryshub.Appearance = {
    Title = "Ryshub",
    Subtitle = "Enter your key to continue",
    Icon = "rbxassetid://95721401302279",
    IconSize = UDim2.new(0, 30, 0, 30)
}
```

### `Ryshub.Links`

```lua
Ryshub.Links = {
    GetKey = "",
    Discord = ""
}
```

Notes:

- in `LaunchPanda()`, `GetKey` is autogenerated if it is still empty
- `Discord` only copies the link to the clipboard
- for this repo, `jsDelivr` is the recommended remote loader

### `Ryshub.Storage`

```lua
Ryshub.Storage = {
    FileName = "Ryshub_Key",
    Remember = true,
    AutoLoad = true
}
```

Rules:

- `LaunchPanda()` can auto-derive `FileName` from `ServiceID`
- do not use `.txt` in `FileName`
- if `.txt` is provided, the core normalizes it
- actual saved file: `Ryshub/keys/<FileName>.txt`

### `Ryshub.Options`

```lua
Ryshub.Options = {
    Keyless = nil,
    KeylessUI = false,
    Blur = true,
    Draggable = true,
    ShowUserPanel = true,
    ShowGetKeyButton = true,
    StreamerMode = false
}
```

Semantics:

- `Keyless`: used by `Launch()`
- `KeylessUI`: used by `Launch()`
- `Blur`: enables background blur
- `Draggable`: allows dragging the UI
- `ShowUserPanel = false`: hides the side user panel and bottom button
- `ShowGetKeyButton = false`: hides `Get Key` and compacts the layout
- `StreamerMode = true`: masks the `player.DisplayName` shown in the user panel

## Callbacks

```lua
Ryshub.Callbacks = {
    OnVerify = nil,
    OnSuccess = nil,
    OnFail = nil,
    OnClose = nil
}
```

Semantics:

- `OnVerify(key)`: only used by `Launch()`
- `OnSuccess()`: recommended; it usually loads the main script
- `OnFail(errorMsg)`: optional
- `OnClose()`: optional

Accepted return types for `OnVerify`:

- `true/false`
- `{ valid = boolean, error = string?, message = string? }`

## Methods

### `Ryshub:Launch()`

Use case:

- custom local validation or your own HTTP validation

Requirements:

- define `Ryshub.Callbacks.OnVerify` if it is not keyless

Behavior:

- supports `Keyless`
- supports `KeylessUI`
- supports autoload of the saved key
- uses `Callbacks.OnVerify` for validation

### `Ryshub:LaunchPanda(config)`

Config:

```lua
{
    ServiceID = "ryshub", -- or config.Service
    FileName = "RysHubPremiumKey", -- optional
    StorageFileName = "RysHubPremiumKey", -- optional alias
    ValidateURL = "https://new.pandadevelopment.net/api/v1/keys/validate", -- optional
    GetKeyURL = "https://new.pandadevelopment.net/getkey/<service>?hwid=<hwid>" -- optional
}
```

Requirements:

- `ServiceID` is required

Behavior:

- builds its own HTTP validation for Panda
- if there is no custom `FileName`, it uses `ServiceID` as the base for the storage file
- auto-generates `Links.GetKey` if it does not exist
- uses autoload if `Storage.AutoLoad = true`
- does not use `Callbacks.OnVerify`
- if `Callbacks.OnSuccess` exists, it runs after validation
- `OnFail` remains optional

Panda request payload:

```json
{
  "ServiceID": "<service>",
  "HWID": "<hwid>",
  "Key": "<key>"
}
```

Panda success detection:

- `data.valid == true`
- `data.Valid == true`
- `data.success == true`
- `data.Authenticated_Status == "Success"`
- `data.Status == "Success"` or `"Authenticated"`

### `Ryshub:LaunchJunkie(config)`

Config:

```lua
{
    Service = "your-service",
    Identifier = "your-identifier",
    Provider = "your-provider"
}
```

Requirements:

- `Service`, `Identifier`, and `Provider` are required

Behavior:

- loads the Junkie SDK from `https://jnkie.com/sdk/library.lua`
- sets `Links.GetKey` from the SDK if it is empty
- uses `Junkie.check_key()` as the validator
- supports automatic keyless detection when `Options.Keyless == nil`
- supports autoload of the saved key
- keeps the original Arqel compatibility behavior where UI-triggered success does not call `OnSuccess` in Junkie mode

## File/State Effects

### Saved key

- save path: `Ryshub/keys/<FileName>.txt`
- save only if:
  - filesystem is supported
  - `Storage.Remember = true`

### Globals

```lua
getgenv().SCRIPT_KEY
getgenv().RyshubLoaded
getgenv().RyshubClosed
getgenv().Ryshub
```

Meaning:

- `SCRIPT_KEY`: validated key or `"KEYLESS"`
- `RyshubLoaded`: prevents loading duplicate UI
- `RyshubClosed`: marks a manual close
- `Ryshub`: global reference to the module

## User Panel Privacy

Current options:

- `ShowUserPanel = false`: removes the entire panel
- `StreamerMode = true`: keeps the panel but masks the name

Current streamer masking:

- uses `player.DisplayName`
- converts it into `*****` or more asterisks depending on length
- does not hide the avatar
- does not hide the HWID
- does not hide executor/device

## External Dependencies

## Recommended Minimal Panda Setup

```lua
local Ryshub = loadstring(game:HttpGet("https://cdn.jsdelivr.net/gh/Ryshub/Ryshub-UI@main/RyshubUi.luau"))()

local function runMainScript()
    print("main script executed")
end

Ryshub.Appearance.Title = "RysHub"

Ryshub.Callbacks.OnSuccess = function()
    runMainScript()
end

Ryshub:LaunchPanda({
    ServiceID = "ryshub"
})
```

Security rule:

- do not present the direct URL of the main payload in a public `OnSuccess` example
- exposing that URL lets users copy the payload location and execute it directly, bypassing the key UI flow
- if payload protection matters, recommend an obfuscation/protection layer such as `Luraph` for the distributed main script
- treat obfuscation as a deterrent that raises effort, not as a perfect security guarantee
